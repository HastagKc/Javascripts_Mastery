Introduction to Datatypes in JS

JavaScript is a dynamically typed language, meaning variables can hold values of any type without explicit type declarations. Data types define the kind of values that can be stored and manipulated. JavaScript categorizes data types into Primitive (pass by value) and Non-Primitive (pass by reference) types.

Types of Data Types

  1. Primitive Data Types (Pass By Value)
    1. Number: Represents numeric values, including integers and floating-point numbers.
    2. String: Represents a sequence of characters. Example: "Hello, world!"
    3. Boolean: Logical data type with only two values: true or false.
    4. null: Represents a deliberate non-value or empty value.
    5. undefined: A variable that has been declared but not assigned a value.
    6. BigInt: Used for very large integers beyond the safe range of Number.
    7. Symbol: Unique and immutable values, often used as object property keys.
  2. Non-Primitive Data Types (Pass By Reference)
    1. Array: An ordered list of values, accessed by index.
    2. Object: A collection of key-value pairs used to store structured data.
    3. Function: Reusable block of code designed to perform a specific task.
    4. Date: Used to work with dates and times.
    5. RegExp: Represents regular expressions used for pattern matching in strings.
    6. Map: Stores key-value pairs where keys can be of any data type.
    7. Set: A collection of unique values with no duplicates.
    8. WeakMap: Like Map, but keys must be objects and are weakly referenced.
    9. WeakSet: Like Set, but only stores objects and they are weakly referenced.
